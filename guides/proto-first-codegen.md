# Proto-First Development with TypewayCodec Codegen

Start from a `.proto` file and generate high-performance Rust types
with `#[derive(TypewayCodec)]`, no manual struct definitions needed.

## When to use proto-first

- Your team defines APIs in `.proto` files first
- You're interoperating with services in other languages
- You receive `.proto` files from another team
- You want a single source of truth for the wire format

## The workflow

### 1. Start with your `.proto` file

```proto
// trading.proto
syntax = "proto3";
package trading.v1;

service OrderBook {
  // POST /orders
  rpc SubmitOrder(Order) returns (OrderAck);
  // POST /cancel
  rpc CancelOrder(CancelRequest) returns (CancelAck);
  // POST /book
  rpc GetOrderBook(SymbolQuery) returns (OrderBookSnapshot);
}

message Order {
  string symbol = 1;
  string side = 2;
  double price = 3;
  uint32 quantity = 4;
}

message OrderAck {
  string order_id = 1;
  string status = 2;
}

message CancelRequest {
  string order_id = 1;
}

message CancelAck {
  string order_id = 1;
  string status = 2;
}

message SymbolQuery {
  string symbol = 1;
}

message OrderBookSnapshot {
  string symbol = 1;
  repeated PriceLevel bids = 2;
  repeated PriceLevel asks = 3;
}

message PriceLevel {
  double price = 1;
  uint32 quantity = 2;
}
```

### 2. Generate Rust types

```rust
// build.rs or a one-time script
fn main() {
    let proto = std::fs::read_to_string("trading.proto").unwrap();
    let rust_code = typeway_grpc::proto_to_typeway_with_codec(&proto).unwrap();
    std::fs::write("src/generated.rs", &rust_code).unwrap();
    println!("Generated src/generated.rs ({} bytes)", rust_code.len());
}
```

### 3. The generated output

```rust
//! Generated from proto package `trading.v1`.
//!
//! This file was auto-generated by `typeway-grpc codegen`.
//! Manual edits may be overwritten.

use typeway::prelude::*;
use serde::{Serialize, Deserialize};
use typeway_macros::{TypewayCodec, ToProtoType};
use typeway_protobuf::BytesStr;

#[derive(Debug, Clone, Default, Serialize, Deserialize, TypewayCodec, ToProtoType)]
pub struct Order {
    #[proto(tag = 1)]
    pub symbol: BytesStr,    // zero-copy decode, automatic!
    #[proto(tag = 2)]
    pub side: BytesStr,
    #[proto(tag = 3)]
    pub price: f64,
    #[proto(tag = 4)]
    pub quantity: u32,
}

#[derive(Debug, Clone, Default, Serialize, Deserialize, TypewayCodec, ToProtoType)]
pub struct OrderAck {
    #[proto(tag = 1)]
    pub order_id: BytesStr,
    #[proto(tag = 2)]
    pub status: BytesStr,
}

// ... more structs ...

// Path type declarations
typeway_path!(type OrdersPath = "orders");
typeway_path!(type CancelPath = "cancel");
typeway_path!(type BookPath = "book");

type API = (
    PostEndpoint<OrdersPath, Json<Order>, Json<OrderAck>>,
    PostEndpoint<CancelPath, Json<CancelRequest>, Json<CancelAck>>,
    PostEndpoint<BookPath, Json<SymbolQuery>, Json<OrderBookSnapshot>>,
);
```

### 4. Write handlers

```rust
mod generated;
use generated::*;

async fn submit_order(body: Json<Order>) -> Json<OrderAck> {
    Json(OrderAck {
        order_id: format!("ORD-{}", rand::random::<u32>()),
        status: "accepted".into(),
    })
}

async fn cancel_order(body: Json<CancelRequest>) -> Json<CancelAck> {
    Json(CancelAck {
        order_id: body.0.order_id,
        status: "cancelled".into(),
    })
}

#[tokio::main]
async fn main() {
    let server = Server::<API>::new((
        bind!(submit_order),
        bind!(cancel_order),
        bind!(get_book),
    ));
    server.serve("0.0.0.0:3000".parse().unwrap()).await.unwrap();
}
```

## Two codegen modes

| Function | Derives | Use when |
|----------|---------|----------|
| `proto_to_typeway()` | Serialize, Deserialize | JSON-only, simple REST |
| `proto_to_typeway_with_codec()` | + TypewayCodec, ToProtoType, Default, BytesStr | Binary gRPC, max performance |

The `_with_codec` version generates:
- `#[derive(TypewayCodec)]` and `#[proto(tag = N)]` for fast binary protobuf
- `BytesStr` instead of `String` for all `string` proto fields, **zero-copy
  decode is automatic**, no manual optimization needed

`BytesStr` eliminates allocation on decode, the decoder slices the
input buffer instead of copying (54% faster than prost on string-heavy
messages).

## Round-trip: Rust-first → proto → codegen

You can also go the other direction, define types in Rust, generate
the `.proto`, then verify the codegen produces equivalent types:

```rust
// Generate .proto from Rust types
let proto = <MyAPI as ApiToProto>::to_proto("MyService", "my.v1");

// Parse and regenerate, the round-trip should be stable
let regenerated = typeway_grpc::proto_to_typeway_with_codec(&proto).unwrap();
```

This is useful for verifying that your Rust types and `.proto` files
stay in sync.
